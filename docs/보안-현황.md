# 보안 현황 요약

## ✅ 이미 적용된 보안

| 영역 | 내용 |
|------|------|
| **관리자 API** | `count-by-school`, `teachers`, `teacher-summaries`, `mileage-by-email`, `points-by-email`, `plan-by-email`, `result-report-by-email`, `diagnosis-upload`, `diagnosis-settings`, `diagnosis-title`, `verify-teacher-email`, `reset-password`, `reset-all-data` — 모두 Bearer 토큰 + 로그인 사용자 확인, 관리자일 때만 본인 학교 데이터만 접근 |
| **AI API** | `ai-recommend`, `ai-classify-mileage`, `ai-summarize-reflections`, `ai-refine-mileage` — 모두 Bearer 필수, 비로그인 시 401 |
| **관리자 코드** | 서버에서 `ADMIN_CODE`만 사용(클라이언트 노출 없음), 입력란은 `type="password"`로 마스킹 |
| **프로필/가입** | `complete-profile` — 관리자 선택 시 학교당 3명 제한을 **서버에서** 검증 (프론트 우회 불가) |
| **RLS (3개 테이블)** | `reflection_drafts`, `user_preferences`, `mileage_entries` — 본인 이메일(`user_email`)과 JWT 이메일이 일치할 때만 접근 가능 (대시보드에서 `rls-security-fix.sql` 실행 필요) |
| **개발용 API** | `check-env` — `NODE_ENV !== "development"` 일 때 404 반환 |

## ⚠️ “완벽”하다고 할 수 없는 부분 (선택 개선)

1. **다른 테이블 RLS**  
   클라이언트에서 anon key로 직접 접근하는 테이블 중 아래는 아직 RLS 스크립트에 포함되지 않았습니다.  
   **현재는 앱 코드에서만 `user_email`로 필터링**하고 있어, 앱을 통한 접근은 본인 데이터만 보이지만, **Supabase anon key를 악용해 다른 사용자 데이터를 조회할 수 있는 이론적 가능성**이 있습니다.  
   - `diagnosis_results`  
   - `development_plans`  
   - `user_points`  
   - (필요 시) `daily_reflections`  
   → 같은 방식으로 `auth.jwt() ->> 'email' = user_email` 정책을 추가하면 더 안전합니다.

2. **학교/관리자 전용 테이블**  
   `school_point_settings`, `school_diagnosis_settings` 등은 학교 단위 데이터라, “해당 학교 관리자/교사만” 접근하도록 RLS나 API 전용 접근으로 제한하는 설계가 있으면 좋습니다. 현재는 주로 **서버 API(인증된 관리자/교사)** 를 통해서만 접근합니다.

3. **인증 코드·비밀번호 재설정**  
   `verify-code`(관리자 코드)는 환경 변수와 비교만 하므로, **코드 유출 시 해당 코드로 관리자 가입이 가능**합니다.  
   `reset-password` 등은 관리자만 호출 가능하도록 이미 제한되어 있습니다.

4. **레이트 리밋**  
   로그인, `verify-code`, `ai-recommend` 등에 **요청 횟수 제한**이 없어, 무차별 대입·남용 가능성이 있습니다.  
   Vercel/Next.js 또는 API Gateway에서 rate limit을 두면 좋습니다.

5. **환경 변수**  
   `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_CODE` 등은 **서버 전용**으로 두고, 클라이언트 번들에 포함되지 않도록 해 두는 것이 좋습니다.  
   `NEXT_PUBLIC_*`가 붙은 값만 클라이언트에 노출됩니다.

## RLS 스크립트 실행 여부

- `supabase/rls-security-fix.sql`은 **앱이 자동 실행하지 않습니다.**  
- 반드시 **Supabase 대시보드 → SQL Editor**에서 내용을 붙여 넣고 **한 번 실행**해 주어야, 위 3개 테이블 RLS가 실제 DB에 적용됩니다.  
- 자세한 절차는 `supabase/RLS-적용-가이드.md`를 참고하세요.
